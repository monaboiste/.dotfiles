# Commit

1. Inspect changes (`git status --short -uall`, `git diff`) and recent commits (`git log -5 --oneline`).
2. Group related files into logical commits.
3. Draft matching commit messages (imperative, ≤50-char subject, 72-char body, prefer lowercase).
4. Show the plan for approval, then stage and commit approved groups.
